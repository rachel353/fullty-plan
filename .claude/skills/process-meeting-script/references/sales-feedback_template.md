# Feedback Template

## Output Location
`meeting_scripts/[MM.DD]/sales-feedback.md`

## Template Structure

```markdown
# Sales Feedback - [Meeting Date] Meeting

## Meeting Overview
- Client: [Client Name]
- Date: [MM.DD]
- Participants: [List of participants]
- Meeting Type: [Consultation / Estimation / Follow-up]

## Speaker Identification
- 리트머스 Team: [List of speakers and their roles]
- Client Team: [List of speakers and their roles]

## Strengths (간단히)
- [단순하고 간단한 긍정적 측면만 몇 가지 bullet로 정리]

## 개선이 필요한 대화 분석

### [주제/카테고리 1]

**상황/시점**: [대화가 발생한 맥락 - 주제, 회의 진행 순서]

**고객사 발언**: "[고객사가 한 정확한 발언 또는 요약]"

**리트머스 응답**: "[리트머스 팀이 한 정확한 발언 또는 요약]"

**문제점**: [이 대화가 왜 문제인지 - 구체적 측면]

**개선 제안**: "[이런 경우에는 이렇게 말했으면 좋았을 것 같다]"

**이유**: [제안한 응답이 왜 더 나은지]

---

### [주제/카테고리 2]
(필요한 만큼 반복)

## 종합 개선 권장사항

### 즉시 적용 가능한 개선사항
- [구체적이고 실행 가능한 액션 아이템]

### 향후 프로세스 개선
- [장기적 개선 방향]
```

## Analysis Guidelines

1. **발언자 식별**
   - 리트머스 팀 멤버 구분
   - 클라이언트 대표 구분

2. **피드백 형식**
   - 고객사 발언 → 리트머스 응답 → 문제점 → 개선 제안 순서
   - 실제 발언 기반 (인용 또는 정확한 요약)

3. **Strengths는 간단히**
   - 단순 bullet 몇 가지만
   - 과도한 칭찬 지양

4. **구체적 예시 필수**
   - 모든 피드백에 정확한 시점/맥락 포함
   - 추상적 피드백 지양
