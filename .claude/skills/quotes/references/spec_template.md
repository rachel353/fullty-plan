# quotes.md Template

## 목표

화면(Screen) 단위로 기능별 견적서(Quote) MD를 생성한다.
**중요** 템플릿에서 벗어난 항목은 추가하지 않는다. 추가가 필요하다고 판단되는 경우 사용자에게 먼저 물어본다. 

## 입력 소스

| 소스 | 추출 정보 |
|------|----------|
| `ia-structure.md` | Screen Hierarchy의 Route, Purpose, Access |
| `user_stories.json` | 해당 Screen에 연결된 US ID, 액션 문장 |
| `project-requirements.md` | 해당 Screen에 연결된 REQ ID, 제목, 설명 |

## 출력 구조

각 Screen은 아래 구조를 반드시 따른다:

1. Screen 제목 (Route 포함)
2. Purpose
3. Access
4. User Stories
5. 기능 목록 및 견적 테이블
6. (추가됨) 항목 안내
7. 견적 요약

## 규칙

1. 견적은 반드시 **REQ 단위**로 작성한다.
2. User Story의 Acceptance Criteria를 충족하는데 REQ가 부족하다고 판단되면:
   - REQ를 임시로 추가한다.
   - 제목에 `(추가됨)`을 붙인다.
   - 견적 테이블의 비고에 `추가됨 / 확인 필요`를 표기한다.
3. 원본 문서의 문구는 최대한 유지하고, 추측이나 과도한 기획 확장은 하지 않는다.

## 주의사항

- ia-structure.md에 없는 Screen은 생성하지 않는다.
- project-requirements.md에 없는 기능은 반드시 `(추가됨)`으로만 생성한다.

---

## (추가됨) 항목 처리 규칙

1. `(추가됨)` 태그는 REQ_NAME 뒤에 붙인다.
2. 견적 테이블 비고란에 `추가됨 / 확인 필요` 표기한다.
3. 별도 섹션에서 해당 항목과 연결된 US ID를 명시한다.
4. 견적 요약에서 "옵션 견적"으로 분리 표시한다.

## 📌 유지보수 및 비용 안내

- 기획/디자인 단계까지 무료 수정 작업 총 2회 이며 사안에 따라서 견적 추가될 수 있습니다. 이후 유지 보수는 별도 비용 청구됩니다.
- 기획 수정을 반영해드릴 경우, 기획/디자인/개발 리소스가 변경되기 때문에 총 프로젝트 기간이 늘어날 수 있습니다.
- 디자인 범위는 UI/UX 디자인을 지원합니다. BX 디자인(예: 로고, IR 등등)은 별도 제작하셔야 합니다.
- 운영에 필요한 서버비는 고객사가 부담하셔야 합니다. 자사 고객사 평균, 통상적으로 초기 오픈 시 5~20만원 사이로 발생됩니다.
- 개발 완료 이후 무료 유지보수 : 본 견적상에 나와있는 기능들 한해 버그 발생시 1개월간 무상보수.
- 유지보수 : 서비스 운영에 필요한 유지보수 및 버그픽스만 해당합니다. 서비스 고도화 및 최적화는 추가개발에 해당합니다. 

**무료 유지보수**
프로젝트 완료 이후에도 고객에게 무료로 제공되는 기본적인 유지보수 서비스입니다. 주로 버그 수정 및 서비스 유지에 필수적인 유지보수가 포함되며 추가 기능개발은 포함되지 않습니다. 프로젝트 규모에 따라 1~2개월의 무료 유지보수 기간을 드립니다.

**유료 유지보수**
(필요할 시만 계약 가능 or 시간제/팀구독 플랜 가능 → 서비스 오픈 전 조율 가능)
무료 유지보수 기간이 끝난 이후에는 버그 수정 및 서비스 유지보수 비용을 유료로 지불해야합니다. 통상적으로 1년 유지보수 비용은 프로젝트 전체 견적의 20% 입니다.

## 제외 항목 (포함하지 않음)

다음 섹션들은 quotes.md에 포함하지 않는다:
- REQ 원본 ID 기준 합계
- 최종 견적 옵션 (Option A/B/C)
- POC 별도 견적
- REQ ID Coverage 검증
- 미확정 요구사항 (UNCONFIRMED)
- Notes
