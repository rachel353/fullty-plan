# quotes.md Template

## 목표

화면(Screen) 단위로 기능별 견적서(Quote) MD를 생성한다.
**중요** 템플릿에서 벗어난 항목은 추가하지 않는다. 추가가 필요하다고 판단되는 경우 사용자에게 먼저 물어본다. 

## 입력 소스

| 소스 | 추출 정보 |
|------|----------|
| `ia-structure.md` | Screen Hierarchy의 Route, Purpose, Access |
| `user_stories.json` | 해당 Screen에 연결된 US ID, 액션 문장 |
| `project-requirements.md` | 해당 Screen에 연결된 REQ ID, 제목, 설명 |

## 출력 구조

각 Screen은 아래 구조를 반드시 따른다:

1. Screen 제목 (Route 포함)
2. Purpose
3. Access
4. User Stories
5. 기능 목록 및 견적 테이블
6. (추가됨) 항목 안내
7. 견적 요약

## 규칙

1. 견적은 반드시 **REQ 단위**로 작성한다.
2. User Story의 Acceptance Criteria를 충족하는데 REQ가 부족하다고 판단되면:
   - REQ를 임시로 추가한다.
   - 제목에 `(추가됨)`을 붙인다.
   - 견적 테이블의 비고에 `추가됨 / 확인 필요`를 표기한다.
3. 원본 문서의 문구는 최대한 유지하고, 추측이나 과도한 기획 확장은 하지 않는다.

## 주의사항

- ia-structure.md에 없는 Screen은 생성하지 않는다.
- project-requirements.md에 없는 기능은 반드시 `(추가됨)`으로만 생성한다.

---

## (추가됨) 항목 처리 규칙

1. `(추가됨)` 태그는 REQ_NAME 뒤에 붙인다.
2. 견적 테이블 비고란에 `추가됨 / 확인 필요` 표기한다.
3. 별도 섹션에서 해당 항목과 연결된 US ID를 명시한다.
4. 견적 요약에서 "옵션 견적"으로 분리 표시한다.

```markdown
## 견적 산정 기준

* 본 견적은 **기능(REQ) 단위**로 산정됩니다.
* `(추가됨)`으로 표시된 항목은
  유저 액션 분석 과정에서 **추가로 정의된 기능**이며,
  **포함 여부 확인 후 최종 견적에 반영됩니다.**
* `(추가됨)` 항목 미포함 시,
  해당 기능과 연결된 **Acceptance Criteria는 조정**됩니다.

## 견적 요약

> ※ 아래 금액은 화면 단위 합산 기준이며,
> 최종 견적은 `(추가됨)` 항목 확정 후 산출됩니다.

| 구분 | 포함 기준 | 포함 기능 | 금액 (VAT 별도) | 비고 |
|------|----------|----------|----------------|------|
| 필수 견적 | 중요도 1 | {{REQ_ID_LIST_P1}} | {{SUM_P1}}만원 | MVP 필수 |
| 권장 견적 | 중요도 1 + 2 | {{REQ_ID_LIST_P1_P2}} | {{SUM_P1_P2}}만원 | |
| 옵션 견적 | (추가됨) | {{REQ_ID_LIST_ADDED}} | {{SUM_ADDED}}만원 | 포함 확인 필요 |

**중요도 배정 기준**

- **중요도 1 (필수)**: 핵심 기능, 없으면 서비스 불가
- **중요도 2 (중요)**: 사용성 향상, UX 개선
- **중요도 3 (선택)**: 부가 기능, 추후 추가 가능

---

## 확인 요청 사항

* `(추가됨)`으로 표시된 기능의 **포함 여부를 확인 부탁드립니다.**
* 확인 결과에 따라:
  * 포함 시 → 최종 견적 반영
  * 미포함 시 → 관련 Acceptance Criteria 조정

> 모든 금액은 VAT 별도입니다.

```

## 제외 항목 (포함하지 않음)

다음 섹션들은 quotes.md에 포함하지 않는다:
- REQ 원본 ID 기준 합계
- 최종 견적 옵션 (Option A/B/C)
- POC 별도 견적
- REQ ID Coverage 검증
- 미확정 요구사항 (UNCONFIRMED)
- Notes
