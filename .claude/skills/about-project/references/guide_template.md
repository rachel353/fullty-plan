(미팅별로 요구사항이 어떻게 변화했는지)

(예산과 일정에 대한 고객사의 우선순위와 결정사항)

## 5) 기술적/기획적 제약사항 및 고려사항

- **요구사항 변화**: 이전 [X]개 → 현재 [Y]개 (신규 추가 [Z]개)
- **적정성 판단**: [✅ 적정 / ⚠️ 부족 / ✅ 여유]

## Section Guidelines

### Section 6: 견적 적정성 평가

**계산 공식:**
- 내부 단가: ₩6,500,000/MM (맨먼스)
- 현실적 견적 = 필요 맨먼스 × ₩6,500,000
- 클라이언트 문서에는 최종 금액만 표시 (MM 수 비공개)

**적정성 판단 기준:**
- ✅ 적정: 현실적 견적 ≤ 협의된 견적
- ⚠️ 부족: 현실적 견적 > 협의된 견적 (범위 조정 필요)
- ✅ 여유: 현실적 견적 < 협의된 견적의 80%

**부족 시 제안 사항:**
1. 높은 견적으로 제시 + 범위 설명
2. 범위 축소안 제시 (선택적 기능 제외)
3. 단계적 개발 제안 (POC → 본개발)

### Section 7: 팀 구성 제안

고려 요소:
- 요구사항 복잡도
- 협의된 일정
- 예산 범위
- 기술 스택 요구사항
