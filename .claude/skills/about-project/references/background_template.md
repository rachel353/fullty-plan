# background.md Template

## Output Structure

```markdown
## 0) 개편안 정리 핵심

(최종 합의 내용을 바탕으로 기존 요구사항과 IA의 핵심 변경 사항들과 협의사항을 정리하고 개편안에 대한 계획과 주안점을 정리. 특히, 기존의 견적 범위가 협의안을 초과하는 경우 견적 범위에 맞출 수 있는 안을 계획하고 이에 맞추어 제거 요구사항을 정리합니다.)

## 1) 프로젝트 개요

(요구사항 기반 핵심 가치·범위 요약)

## 2) 서비스 배경 및 필요성

(해결하려는 비즈니스 Pain Point, 시장·운영 상의 문제, 도입 필요성)

## 3) 프로젝트 주요 목표

(시스템 구축을 통해 달성해야 하는 비즈니스 성과 목표)

## 4) 개발 범위 및 결과물

(제공될 기능 범위, 산출물, 인수인계 범위, 운영 단계 포함 여부)

## 5) 유의할 기술적/기획적 포인트

(아키텍처 제약, 외부 연동 리스크, 성능 요구사항, 운영·보안 고려사항)
```

## Section Guidelines

### Section 0: 개편안 정리 핵심

이 섹션은 **기존 견적/협의안이 있는 경우**에만 작성합니다.

포함 내용:
- 기존 협의 대비 변경된 핵심 사항
- 견적 범위 초과 시 조정안
- 제거/연기된 요구사항 목록
- 개편의 주안점 및 방향성

### Section 1: 프로젝트 개요

작성 원칙:
- 2-3문장으로 핵심 가치 요약
- 주요 사용자와 핵심 기능 언급
- 비즈니스 임팩트 명시

### Section 2: 서비스 배경 및 필요성

포함 요소:
- 현재 문제점 (Pain Point)
- 시장/경쟁 상황
- 도입 시 기대 효과

### Section 3: 프로젝트 주요 목표

목표 예시:
- 사용자 경험 개선
- 브랜드 가치 향상
- 업무 효율성 증대
- 서비스 품질 향상

### Section 4: 개발 범위 및 결과물

명시 항목:
- 포함 기능 (In-Scope)
- 제외 기능 (Out-of-Scope)
- 산출물 목록
- 인수인계 범위

### Section 5: 기술적/기획적 포인트

고려 사항:
- 기술 스택 제약
- 외부 시스템 연동
- 성능 요구사항
- 보안/규정 준수
