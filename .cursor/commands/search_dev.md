# Search Dev

## Overview
Perplexity를 활용하여 GitHub과 Reddit에서 개발 관련 정보를 검색하고, 시니어 개발자 입장에서 구체적인 개발 방식과 가장 최신의 커뮤니티 노하우 및 의견을 정리하여 제공합니다.

## Steps
1. 검색 목적 파악
    사용자가 요청한 개발 주제나 문제 파악
    시니어가 주니어에게 전달할 핵심 내용 식별
2. 검색 실행
    공식 개발자 문서를 최우선으로 참고하여 검색 진행
    Google이나 Perplexity 등 다양한 검색 방법 사용 가능 (특정 도구에 한정하지 않음)
        검색 시, 공식 레퍼런스 문서, nodejs.org, react.dev, docs.python.org 등의 신뢰할 수 있는 개발자 사이트를 우선적으로 확인
        부가적으로 GitHub, 오픈소스 레포지토리, Reddit 등 커뮤니티·실제 코드·토론 자료도 참고
        오픈소스 활용시 star 수, 최근 커밋, 이슈 등 신뢰도 기준을 반드시 확인
        검색어에 site:github.com, site:reddit.com 및 공식문서 도메인 등을 명시적으로 포함
3. 정보 수집 및 정리
    GitHub: 실제 코드 예시, 베스트 프랙티스, 이슈 토론
        Star 수 기반 가중치 적용 필수
        10,000+ stars: 매우 높은 가중치 (업계 표준, 최우선 참조)
        5,000-9,999 stars: 높은 가중치 (인기 프로젝트, 우선 참조)
        1,000-4,999 stars: 중간 가중치 (참조 가능, 추가 검증 권장)
        100-999 stars: 낮은 가중치 (주의 필요, 코드 리뷰 필수)
        100 미만 stars: 매우 낮은 가중치 (최후 수단, 철저한 검토 필요)
        Star 수가 높은 저장소를 우선적으로 검색 결과에 포함
        여러 저장소가 있을 경우 star 수 순으로 정렬하여 제시
        Star 수 외에도 최근 커밋 활동, 이슈 해결율, 문서화 품질도 함께 고려
    Reddit: 개발자 커뮤니티 경험, 실무 팁, 토론 내용
    Hacker News: 업계 최신 동향, 실무적인 트렌드, 개발자들이 관심을 갖는 주요 논의와 커뮤니티 반응
        주요 관련 이슈나 기술이 Hacker News에서 주목을 받았는지 확인
        높은 upvote, 활발한 토론, 신뢰성 있는 코멘트 및 구체적 경험 사례를 우선 참고
        "site:news.ycombinator.com" 등 명시적 도메인 검색 활용
    최신 정보와 검증된 방법을 GitHub, Reddit, Hacker News 등 다양한 소스에서 우선적으로 수집
4. 구체적인 가이드 작성
    코드 예시와 설명 포함
    실무에서 바로 적용 가능한 형태로 구성
    시니어 관점에서 주의사항과 팁 추가
5. 검증 및 보완
    수집한 정보의 정확성 확인
    추가로 필요한 정보가 있으면 보완 검색
    최종 가이드 완성

# Checklist
    GitHub과 Reddit에서 충분한 정보 수집
    GitHub 저장소의 star 수를 확인하고 가중치를 적용하여 우선순위 결정
    Star 수가 높은 저장소를 우선적으로 참조 및 링크 제공
    주니어 개발자가 이해하기 쉽게 작성
    구체적인 예시와 설명 포함
    실무 적용 가능한 형태로 정리
    시니어 관점의 팁과 주의사항 포함
