# Save Attached File

## Overview
첨부된 파일을 저장하고 파일 형식에 따라 자동 처리합니다. 미팅 스크립트의 경우 자동으로 요구사항을 생성하고, 이미지/문서 파일의 경우 Gemini Vision으로 분석하여 상세 설명을 생성합니다. **최대한 많은 내용을 담을 수 있도록 하며, 파일을 그대로 쓸 수 있으면 원문 그대로 가져오는 것을 목표로 합니다.**

## Steps
1. 첨부된 파일과 대상 경로 확인
2. 터미널 복사 명령어 사용하여 파일 저장 (파일 내용 출력 금지)
3. 파일 형식 확인:
   - **미팅 스크립트** (`script.md`): 
     - 클라이언트 이름과 미팅 날짜 추출
     - `process_meeting/main` 커맨드 자동 실행
     - `process_meeting/requirements` 커맨드 자동 실행
   - **이미지/문서** (`.pptx`, `.docx`, `.pdf`, `.png` 등):
     - 텍스트 및 이미지 추출 스크립트 실행
     - Gemini Vision API로 이미지 분석
     - **원문 그대로 사용 가능한 경우 원문 그대로 가져오기 (최우선)**
     - **최대한 많은 내용을 포함하도록 처리**
     - **파일이 길면 나눠서 작업하고 최종적으로 합쳐서 마무리**
     - 통합 문서 생성
4. `git status` 확인 → `git add` → `git commit`

## 처리 원칙

1. **원문 보존 우선**: 파일을 그대로 쓸 수 있으면 원문 그대로 가져오는 것을 목표로 합니다.
2. **최대 내용 포함**: 최대한 많은 내용을 담을 수 있도록 처리합니다.
3. **긴 파일 분할 처리**: 파일이 길면 나눠서 작업하고, 각 부분을 처리한 후 최종적으로 합쳐서 마무리합니다.
4. **원문과 분석 병행**: 원문을 보존하면서 필요한 경우 분석 내용을 추가로 포함합니다.

- [ ] 원문 사용 가능 시 원문 그대로 가져왔는가?
- [ ] 최대한 많은 내용이 포함되었는가?

- [ ] 미팅 스크립트인 경우 자동 처리가 실행되었는가?
- [ ] 이미지/문서인 경우 분석이 완료되었는가?
