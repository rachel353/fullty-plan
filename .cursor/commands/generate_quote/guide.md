# Generate Quote Guide

## Overview
모든 이전 미팅을 분석하여 클라이언트의 핵심 우선순위, 중요 합의사항, 주요 고려사항을 정리한 가이드 문서를 생성합니다. 이후 모든 견적서 생성 작업의 기초가 됩니다.

## Steps
1. 클라이언트 폴더 찾기 (`YY_MM_CLIENT_NAME/`)
2. `quotes/[MM.DD]/note.md` 확인 (존재 시 최우선)
3. 모든 미팅 요약 읽기:
   - 모든 `meeting_scripts/[MM.DD]/summary.md` 파일 (시간순)
   - 최신 `meeting_scripts/[MM.DD]/requirements.md`
   - 최신 `meeting_scripts/[MM.DD]/script.md` (선택)
   - `appendix/requirements.md` (존재 시 필수)
4. 분석 수행:
   - 클라이언트 우선순위 식별
   - 요구사항 진화 추적
   - 중요 합의사항 추출
   - 견적 적정성 평가 (내부 계산용)
5. `quotes/[MM.DD]/guide.md` 생성 (기존 파일 덮어쓰기)

## Output Format

파일은 다음 구조를 따라야 합니다:

```markdown
# Quote Generation Guide - [Client Name] - [Meeting Date]

## 1) 고객사 핵심 가치 및 우선순위

(고객사가 여러 미팅에서 반복적으로 강조하거나 중요하게 생각하는 것들)

## 2) 중요 합의사항 요약

(모든 미팅을 통틀어 가장 중요한 합의사항들)

## 3) 요구사항 진화 과정

(미팅별로 요구사항이 어떻게 변화했는지)

## 4) 예산 및 일정 관련 중요 결정사항

(예산과 일정에 대한 고객사의 우선순위와 결정사항)

## 5) 기술적/기획적 제약사항 및 고려사항

(고객사가 강조한 기술적 제약이나 기획적 고려사항)

## 6) 견적 적정성 평가 및 범위 조정 제안

- **협의된 견적**: [X]원 (summary.md Section 2)
- **현실적인 견적**: [Y]원 ([X]맨먼스 × ₩6,500,000)
- **요구사항 변화**: 이전 [X]개 → 현재 [Y]개 (신규 추가 [Z]개)
- **적정성 판단**: [✅ 적정 / ⚠️ 부족 / ✅ 여유]
- **제안 사항**: 견적 전략 및 증가/조정 내역

## 7) 프로젝트 팀 구성 및 일정 최적안

(요구사항, 예산, 합의된 일정을 고려한 최적의 팀 구성 및 일정 제안)
```

## Checklist
- [ ] note.md가 읽혔는가? (존재 시)
- [ ] 모든 미팅 요약이 읽혔는가?
- [ ] appendix/requirements.md가 읽혔는가? (존재 시)
- [ ] 클라이언트 우선순위가 식별되었는가?
- [ ] 견적 적정성이 평가되었는가?
- [ ] guide.md가 올바른 구조로 생성되었는가?
