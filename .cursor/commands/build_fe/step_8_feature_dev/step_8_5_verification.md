# 8-5. 동작 확인

## Overview
현재 기능이 “사용자 관점에서” 동작하는지 확인합니다. 결과는 다음 기능으로 넘어갈 수 있도록 **검증 리포트**로 남깁니다.

## Steps
1. 렌더링 확인: 기능에 포함된 주요 페이지(라우트)가 에러 없이 뜨는지 확인합니다.
2. User Flow 테스트: 8-2에서 정한 Flow를 실제로 따라가며 “예상/실제”를 기록합니다.
3. AC 검증: 8-1의 AC별로 “구현/동작 확인/Mock 여부”를 정리합니다.
4. 반응형/브라우저(필요 시) 확인을 간단히 체크합니다.
5. `docs/features/FXX_기능명_verification.md`를 작성/갱신합니다.
6. (변경사항이 생긴 경우) `git status` 확인 → `git add` → `git commit`

## Checklist
- [ ] 핵심 페이지가 모두 렌더링되는가?
- [ ] User Flow가 끊기지 않고 진행되는가?
- [ ] AC가 전부 결과로 매핑되어 있는가?
- [ ] 주요 화면에서 콘솔 에러가 없는가?
- [ ] (변경사항이 생긴 경우) 커밋까지 완료했는가?
