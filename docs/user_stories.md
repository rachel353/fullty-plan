# User Stories - 쟈근친구들 V2 (Conture)

## 프로젝트 개요

**프로젝트명:** 쟈근친구들 V2 (Conture)
**설명:** 작가가 스토리 초안(콘티)을 올리면, 팬들이 창작에 참여하고 기여를 크레딧으로 인정받는 참여형 플랫폼
**버전:** 2.0.0

---

## Actors (역할 정의)

### 1. 독자 (Reader/Backer)
- **설명:** 작품을 탐색하고, 후원/제안을 통해 창작에 참여하는 일반 사용자
- **권한:**
  - 작품/에피소드 열람
  - Ink 충전 및 보유
  - 제안(Proposal) 생성
  - 제안 지지(Backing)
  - 직접 후원(Direct Backing)
  - 본인 기여 내역 조회
- **목표:**
  - 원하는 작품 탐색 및 열람
  - 스토리 전개에 개입하여 창작 참여
  - 크레딧에 이름이 기록되어 인정받기

### 2. 작가 (Creator)
- **설명:** 콘텐츠를 제작하고, 제안을 관리하며, 수익을 정산받는 창작자
- **권한:**
  - 에피소드 업로드 및 관리
  - 제안 Deal/Drop 결정
  - 정산금 조회 및 출금 요청
  - 본인 작품 관리
- **목표:**
  - 제작비 확보
  - 독자 반응 선행 수집
  - 창작물에 대한 수익 정산

### 3. 관리자 (Admin)
- **설명:** 플랫폼 운영, 정산 승인, 콘텐츠 관리를 담당하는 운영자
- **권한:**
  - 전체 회원 관리 (CRUD)
  - 전체 콘텐츠 관리 (숨김/삭제)
  - 정산 요청 승인/거절
  - 유저 제재
  - 시스템 설정 관리
- **목표:**
  - 플랫폼 안정적 운영
  - 유해 콘텐츠 관리
  - 정산 처리 및 수수료 관리

## User Stories by Priority

### P1 - Must Have (MVP 필수)

#### 인증/회원

**US-001** `P1`
As a 독자, I want to 이메일 또는 소셜 계정으로 회원가입하다, So that 서비스를 이용하고 Ink를 소유할 수 있다

**Acceptance Criteria:**
- AC-1: 이메일/비밀번호 입력 시 비밀번호 규칙 실시간 체크 (8자 이상, 영문, 숫자, 특수문자(선택)) (change-2026-01-14-1500)
- AC-2: 비밀번호 확인 필드에서 불일치 시 즉시 에러 상태 표시 (change-2026-01-14-1500)
- AC-3: 휴대폰 번호 입력 후 '인증번호 전송' 버튼 클릭 시 인증번호 발송 및 타이머 시작 (5분) (change-2026-01-14-1500)
- AC-4: 인증번호 일치 시 회원가입 완료 버튼 활성화 (change-2026-01-14-1500)
- AC-5: 카카오/구글 소셜 로그인 버튼 클릭 시 해당 OAuth 인증 페이지로 리다이렉트
- AC-6: 회원가입 완료 후 완료 페이지 표시 → 로그인 CTA 제공 (change-2026-01-14-1500)
- AC-7: 에러 메시지는 필드 하단에 인라인 텍스트로 노출 (change-2026-01-14-1500)

---

**US-002** `P1`
As a 독자, I want to 로그인하다, So that 내 지갑과 기여 내역에 접근할 수 있다

**Acceptance Criteria:**
- AC-1: 올바른 이메일/비밀번호 입력 시 JWT 토큰 발급 및 홈 화면으로 이동
- AC-2: 잘못된 비밀번호 입력 시 '비밀번호가 일치하지 않습니다' 에러 표시
- AC-3: 소셜 로그인 성공 시 3초 이내 홈 화면으로 이동
- AC-4: 비밀번호 찾기 버튼 클릭 시 비밀번호 재설정 페이지로 이동 (change-2026-01-14-1500)

---

**US-002-1** `P1` (change-2026-01-14-1500)
As a 독자, I want to 비밀번호를 재설정하다, So that 비밀번호를 잊어버렸을 때 계정에 다시 접근할 수 있다

**Acceptance Criteria:**
- AC-1: 휴대폰 번호 입력 후 '인증번호 전송' 클릭 시 SMS 발송 및 타이머 시작 (5분)
- AC-2: 인증번호 일치 시 새 비밀번호 설정 화면으로 이동
- AC-3: 새 비밀번호 입력 시 동일한 비밀번호 규칙 실시간 체크
- AC-4: 비밀번호 재설정 완료 시 로그인 페이지로 이동

---

**US-002-2** `P2` (change-2026-01-14-1500)
As a 독자, I want to 회원 탈퇴하다, So that 더 이상 서비스를 이용하지 않을 때 계정을 삭제할 수 있다

**Acceptance Criteria:**
- AC-1: 마이페이지 → 계정 설정에서 '회원 탈퇴' 버튼 제공
- AC-2: 탈퇴 유의사항 안내 박스 표시 (개인정보 삭제, 포인트 소멸, 복구 불가 등)
- AC-3: 현재 비밀번호 입력 + 확인 문구 '회원탈퇴' 정확히 입력 + 동의 체크박스 모두 충족 시 탈퇴 버튼 활성화
- AC-4: 탈퇴 완료 시 로그아웃 및 로그인 페이지로 이동

#### 지갑/재화

**US-003** `P1`
As a 독자, I want to 내 Ink 잔액을 확인하다, So that 후원/제안에 사용 가능한 금액을 알 수 있다

**Acceptance Criteria:**
- AC-1: 마이페이지 '내 지갑' 탭에서 Ink 잔액 정보를 구분하여 표시
  - 사용 가능한 Ink (Available Ink): 즉시 사용할 수 있는 잔액
  - 진행 중인 제안 Ink (Pending Ink): Proposal 및 Backing으로 에스크로에 묶여있는 금액 합계
  - 총 보유 Ink (Total Ink): Available + Pending의 합계
- AC-2: 잔액은 소수점 없이 정수로 표시
- AC-3: 페이지 로드 시 2초 이내 잔액 정보 표시
- AC-4: Pending Ink 영역은 클릭 가능한 UI로 표시 (화살표 아이콘 또는 터치 가능 표시)

---

**US-003-1** `P1`
As a 독자, I want to 진행 중인 제안에 묶인 Ink 내역을 확인하다, So that 어떤 에피소드에 얼마를 제안/지지했는지 파악할 수 있다

**Acceptance Criteria:**
- AC-1: '내 지갑'에서 Pending Ink 영역 클릭 시 '진행 중인 제안 내역' 화면으로 이동
- AC-2: 내역 리스트에 다음 정보 표시:
  - 작품명 / 에피소드 제목
  - 제안 유형 (내가 생성한 Proposal / 내가 지지한 Backing)
  - 베팅/지지 금액
  - 제안 상태 (진행 중 / 마감 임박)
  - 제안 마감일
- AC-3: 리스트 항목 클릭 시 해당 에피소드의 제안 상세 화면으로 이동
- AC-4: 빈 내역일 경우 '진행 중인 제안이 없습니다' 안내 메시지 표시
- AC-5: 페이지 로드 시 2초 이내 내역 리스트 표시

---

**US-003-2** `P2`
As a 독자, I want to 완료된 제안의 Ink 정산 내역을 확인하다, So that 채택/미채택 결과와 환불/보상 내역을 알 수 있다

**Acceptance Criteria:**
- AC-1: '내 지갑'에서 '정산 내역' 탭 또는 필터로 완료된 제안 확인 가능
- AC-2: 내역 리스트에 다음 정보 표시:
  - 작품명 / 에피소드 제목
  - 제안 결과 (채택됨 / 미채택)
  - 원래 베팅/지지 금액
  - 정산 결과 (환불액 / 보상액 / 작가에게 전달된 금액)
  - 정산 완료일
- AC-3: 리스트 항목 클릭 시 해당 에피소드의 제안 결과 상세 화면으로 이동

As a 독자, I want to 홈에서 추천/인기 작품을 확인하다, So that 관심 있는 작품을 쉽게 발견할 수 있다

#### 에피소드 뷰어

**US-008** `P1`
As a 독자, I want to 에피소드 콘티를 열람하다, So that 스토리 내용을 확인하고 참여 여부를 결정할 수 있다

**Acceptance Criteria:**
- AC-1: 콘텐츠 뷰어에서 이미지/텍스트 스크롤 방식으로 표시

- AC-3: PUBLISHED 상태 에피소드에서 액션 바 비노출, 읽기 전용 모드

---

**US-012** `P1`
As a 독자, I want to 완성된 에피소드에서 크레딧을 확인하다, So that 내 기여가 기록되었는지 확인하고 성취감을 느낄 수 있다

**Acceptance Criteria:**
- AC-1: 에피소드 최하단에 [Sponsored by] 크레딧 섹션 표시
- AC-2: 기여 금액 상위 3인 Main Producer로 별도 UI 강조
- AC-3: 채택된 제안자 우선 표시 후 금액순 정렬
- AC-4: 크레딧 데이터는 영구 저장되어 변경 불가

#### 스토리 인터벤션

**US-009** `P1`
As a 독자, I want to 제안(Proposal)을 생성하다, So that 스토리 전개에 개입하고 채택 시 크레딧에 이름을 남길 수 있다

**Acceptance Criteria:**
- AC-1: 제안 작성 모달에서 내용 입력 및 초기 베팅액(최소 100 Ink) 설정
- AC-2: 베팅액이 보유 Ink보다 많을 경우 '잔액이 부족합니다' 에러 표시
- AC-3: 제안 생성 시 즉시 Ink 차감 및 에스크로 처리
- AC-4: 제안 생성 완료 시 '제안이 등록되었습니다' 확인 메시지 표시

---

**US-010** `P1`
As a 독자, I want to 다른 사람의 제안에 지지(Backing)하다, So that 해당 제안의 채택 확률을 높이고 크레딧에 참여할 수 있다

**Acceptance Criteria:**
- AC-1: 제안 상세 카드에서 지지하기 버튼 및 금액 입력 가능
- AC-2: 지지 시 즉시 Ink 차감 및 에스크로 처리
- AC-3: 지지 완료 후 해당 제안의 total_bounty, backer_count 실시간 업데이트 표시

---

**US-011** `P1`
As a 독자, I want to 직접 후원(Direct Backing)하다, So that 제안 없이 순수하게 창작을 지원하고 크레딧에 이름을 남길 수 있다

**Acceptance Criteria:**
- AC-1: 후원 팝업에서 금액 선택 가능
- AC-2: 후원 시 에스크로 없이 즉시 작가 정산 예정금으로 이동
- AC-3: 후원 완료 시 '후원이 완료되었습니다' 확인 메시지 표시

#### 크리에이터 스튜디오

**US-014** `P1`
As a 작가, I want to 에피소드 콘티를 업로드하다, So that 팬들의 후원과 제안을 받을 수 있다

**Acceptance Criteria:**
- AC-1: 저작 에디터에서 이미지 다중 업로드 및 순서 변경 가능
- AC-2: 텍스트 입력 영역 제공
- AC-3: 업로드 완료 후 에피소드 상태 OPEN으로 자동 변경
- AC-4: 업로드 완료 시 '에피소드가 등록되었습니다' 확인 메시지 표시

---

**US-015** `P1`
As a 작가, I want to 에피소드 목록을 관리하다, So that 내 콘텐츠 현황을 파악하고 관리할 수 있다

**Acceptance Criteria:**
- AC-1: 에피소드 관리 리스트에서 각 에피소드의 상태(OPEN/PUBLISHED) 표시
- AC-2: 에피소드별 총 모금액 표시
- AC-3: 에피소드 편집 페이지로 이동 가능

---

**US-019** `P1`
As a 작가, I want to 완성 원고를 업로드하다, So that 에피소드를 공개하고 정산을 받을 수 있다

**Acceptance Criteria:**
- AC-1: 최종 원고 업로드 시 에피소드 상태 PUBLISHED로 변경
- AC-2: 업로드 시 다이내믹 크레딧 자동 생성
- AC-3: 예정 정산금이 출금 가능 금액으로 전환(수수료 차감)
- AC-4: 업로드 완료 시 '원고가 공개되었습니다' 확인 메시지 표시

---

#### 제안 관리

**US-016** `P1`
As a 작가, I want to 받은 제안 목록을 조회하다, So that 어떤 제안이 있는지 확인하고 채택 여부를 결정할 수 있다

**Acceptance Criteria:**
- AC-1: 제안 관리 대시보드에서 모금액 순 정렬된 제안 목록 표시
- AC-2: 각 제안에 내용, total_bounty, backer_count 표시
- AC-3: 제안별 Deal/Drop 액션 버튼 제공

---

**US-017** `P1`
As a 작가, I want to 제안을 채택(Deal)하다, So that 해당 제안의 Bounty를 정산 예정금으로 받을 수 있다

**Acceptance Criteria:**
- AC-1: Deal 버튼 클릭 시 확인 모달 표시
- AC-2: 확인 후 Proposal 상태 DEAL로 변경
- AC-3: 에스크로 된 Ink가 작가 payable_balance로 이동
- AC-4: Deal 완료 시 '제안이 채택되었습니다' 확인 메시지 표시

---

**US-018** `P1`
As a 작가, I want to 제안을 거절(Drop)하다, So that 원하지 않는 제안을 거절하고 참여자에게 환불할 수 있다

**Acceptance Criteria:**
- AC-1: Drop 버튼 클릭 시 확인 모달 표시
- AC-2: 확인 후 Proposal 상태 DROP으로 변경
- AC-3: 해당 Proposal의 모든 Backer에게 Ink 100% 자동 반환
- AC-4: Drop 완료 시 '제안이 거절되었습니다' 확인 메시지 표시

---

#### 정산

**US-020** `P1`
As a 작가, I want to 정산 현황을 조회하다, So that 예정 정산금과 출금 가능 금액을 확인할 수 있다

**Acceptance Criteria:**
- AC-1: 정산 페이지에서 예정 정산금 금액 표시
- AC-2: 출금 가능 금액 별도 표시
- AC-3: 에피소드별 정산 내역 리스트 제공

**Acceptance Criteria:**

- AC-4: 작품 목록의 각 항목 클릭 시 해당 작품 상세 페이지로 이동
- AC-5: 작품이 없을 경우 '등록된 작품이 없습니다' 안내 메시지 표시
- AC-6: 페이지 로드 시 2초 이내 작가 정보 및 작품 목록 표시

#### 신고

**US-029** `P2`
As a 독자/작가, I want to 다른 사용자 또는 콘텐츠를 신고하다, So that 부적절한 행동이나 콘텐츠를 관리자에게 알릴 수 있다

**Acceptance Criteria:**
- AC-1: 스토리 상세, 제안 카드(ProposalCard), 사용자 프로필 등에서 '신고하기' 버튼 제공
- AC-2: 신고 모달에서 신고 유형 선택 가능 (스팸, 욕설/비방, 부적절한 콘텐츠, 저작권 침해, 허위 정보, 기타) - 총 6가지
- AC-3: 신고 사유 상세 입력 필수 (최소 10자 이상)
- AC-4: 동일 대상에 대한 중복 신고 시 '이미 신고한 사용자입니다' 안내 표시
- AC-5: 신고 완료 시 '신고가 접수되었습니다' 확인 메시지 표시
- AC-6: 작가는 본인 스토리에 달린 Proposal 작성자만 신고 가능 (change-2026-01-12-1530)

---

**US-030** `P2`

- AC-3: 처리 상태: 접수됨, 검토 중, 처리 완료, 기각
- AC-4: 신고 항목 클릭 시 상세 내용 및 관리자 피드백 확인 가능

- AC-2: 처리 상태별 필터 제공 (전체, 접수됨, 검토 중, 처리 완료, 기각)
- AC-3: 신고 유형별 필터 제공
- AC-4: 최신순/오래된순 정렬 제공

**US-032** `P2`
As a 관리자, I want to 신고를 처리하다, So that 신고 내용을 검토하고 적절한 조치를 취할 수 있다

**Acceptance Criteria:**
- AC-1: 신고 상세에서 신고자 정보, 피신고자 정보, 신고 사유 상세 확인 가능
- AC-2: 피신고자의 과거 신고 이력 및 제재 이력 표시
- AC-3: 처리 액션 선택 가능: 기각, 경고 부여, 일시 정지, 영구 정지
- AC-4: 처리 시 관리자 메모 입력 가능 (신고자에게 공개되는 피드백과 내부 메모 구분)
- AC-5: 처리 완료 시 신고자에게 처리 결과 알림 발송 (선택적)
- AC-6: 제재 조치 선택 시 US-023 제재 프로세스와 연동

#### 설정 (Settings)

**US-033** `P2`
As a 독자/작가, I want to 내 프로필 정보를 수정하다, So that 닉네임과 프로필 이미지를 원하는 대로 변경할 수 있다

**Acceptance Criteria:**
- AC-1: Settings 페이지에서 현재 프로필 이미지 및 닉네임 표시
- AC-2: 프로필 이미지 업로드 시 JPG/PNG/WEBP 형식만 허용, 최대 3MB 제한
- AC-3: 이미지 업로드 전 미리보기 제공
- AC-4: 닉네임 입력 시 2~12자, 한글/영문/숫자/`_`/`.`만 허용
- AC-5: 닉네임 입력 중 실시간 중복 확인 (500ms debounce)
- AC-6: 중복 닉네임 입력 시 '이미 사용 중인 닉네임입니다' 에러 표시
- AC-7: 저장 성공 시 '프로필이 업데이트되었습니다' 메시지 표시 및 authStore 동기화
- AC-8: 기본 정보(닉네임/이미지) 수정 시 비밀번호 재확인 불필요
- AC-9: (작가 전용) 작가소개글 편집 UI 제공 (Plain Text Textarea, 500~1000자 제한)
- AC-10: (작가 전용) 작가소개글 입력 중 실시간 글자 수 카운터 표시
- AC-11: (작가 전용) 작가소개글 저장 성공 시 authStore 동기화
