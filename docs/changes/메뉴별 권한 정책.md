# 메뉴별 권한 정책

# **📌 메뉴별 권한 정책 (정리본 · LLM Friendly)**

## **0. 공통 원칙**

- **역할(Role)**: C-level, GA, Manager, PM, Sales, Designer, Developer, QA
- **역할은 중복 선택 가능**
    - 하나의 사용자에게 여러 역할(Role)이 동시에 부여될 수 있음
    - 이 경우 접근 가능 메뉴, 버튼, 데이터 노출 권한은 각 역할이 가진 권한의 합집합으로 적용된다.
    (즉, 하나의 역할이라도 허용한 권한은 최종적으로 허용됨)
- **실제 비용(Actual Cost)은 전 역할에서 노출 금지**
- **모든 비용 계산은 “레벨 비용(Level Cost)” 기준**
- 버튼 권한과 데이터 노출 권한은 **독립적으로 관리**

## **6. 핵심 보안 정책 요약 (LLM Rule용)**

```
- actual_cost: always hidden
- cost_calculation: level_cost only
- employee_salary: only GA can see raw salary
- financial_actions: GA initiates, C-level/Manager/PM/Sales finalize
- workforce_my_tasks: individual contributors only
- settings_access: C-level, GA only
```
