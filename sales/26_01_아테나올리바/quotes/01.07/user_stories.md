# User Stories - 아테나올리바

> 작성일: 2026-01-07
> 총 User Story: 20개 (P1: 14, P2: 6, P3: 0)

---

## Actors (역할 정의)

### 일반 사용자
- **설명**: 올리브오일을 스캔하고 구매하는 소비자
- **권한**: 라벨 스캔, 상품 조회, 장바구니, 결제, 리뷰 작성
- **목표**: 올리브오일 정보 빠르게 확인, 신뢰할 수 있는 정보 기반 구매 결정, 편리한 구매 경험

### 운영 관리자
- **설명**: 상품 및 라벨 DB를 관리하는 아테나올리바 운영 담당자
- **권한**: 상품 CRUD, 라벨 데이터 CRUD, 회원 관리, 주문 관리
- **목표**: 상품 정보 정확하게 관리, 라벨 DB 최신 상태 유지, 회원 및 주문 현황 파악

### 전문가 리뷰어
- **설명**: 올리브오일 전문 리뷰를 작성하는 전문가
- **권한**: 전문가 리뷰 작성, 상품 정보 조회
- **목표**: 전문적인 올리브오일 평가 제공, 소비자 구매 결정 도움

---

## P1 - Must Have (14개)

### 일반 사용자

#### US-101: 올리브오일 라벨 스캔 [REQ-101]
> As a **일반 사용자**, I want to **올리브오일 라벨을 카메라로 촬영하다**, So that **제품 정보를 즉시 확인할 수 있다**

**Acceptance Criteria:**
- AC-1: 카메라 권한 요청 시 '카메라 접근 권한이 필요합니다' 메시지 표시
- AC-2: 라벨 촬영 후 3초 이내에 제품 정보 또는 '제품을 찾을 수 없습니다' 메시지 표시
- AC-3: 스캔 중 로딩 인디케이터 표시
- AC-4: 스캔 결과로 제품명, 원산지, 가격이 표시됨

---

#### US-102: 스캔 결과 매칭 [REQ-103]
> As a **일반 사용자**, I want to **스캔된 라벨과 가장 유사한 제품을 추천받다**, So that **정확한 제품을 찾을 수 있다**

**Acceptance Criteria:**
- AC-1: 정확히 매칭되는 제품이 있으면 해당 제품 상세 페이지로 이동
- AC-2: 정확한 매칭이 없으면 유사도 높은 상위 3개 제품 목록 표시
- AC-3: 각 추천 제품에 유사도 점수 또는 '추천' 레이블 표시
- AC-4: '제품이 없습니다' 상태에서 '새 제품 요청' 버튼 제공

---

#### US-103: 상품 상세 정보 조회 [REQ-104]
> As a **일반 사용자**, I want to **스캔된 제품의 상세 정보를 확인하다**, So that **구매 결정을 내릴 수 있다**

**Acceptance Criteria:**
- AC-1: 제품명, 가격, 원산지, 품질 등급이 표시됨
- AC-2: 제품 이미지가 표시됨 (이미지 없으면 기본 이미지)
- AC-3: '장바구니 담기' 버튼이 표시됨
- AC-4: 리뷰 평점 평균과 리뷰 개수가 표시됨

---

#### US-104: 장바구니 담기 [REQ-105]
> As a **일반 사용자**, I want to **상품을 장바구니에 담다**, So that **여러 상품을 한 번에 구매할 수 있다**

**Acceptance Criteria:**
- AC-1: '장바구니 담기' 버튼 클릭 시 '장바구니에 담겼습니다' 토스트 메시지 표시
- AC-2: 장바구니 아이콘에 담긴 상품 개수 뱃지 표시
- AC-3: 장바구니에서 수량 변경 가능 (1-99개)
- AC-4: 장바구니에서 상품 삭제 가능

---

#### US-105: 결제 [REQ-105]
> As a **일반 사용자**, I want to **장바구니 상품을 결제하다**, So that **올리브오일을 구매할 수 있다**

**Acceptance Criteria:**
- AC-1: 결제 버튼 클릭 시 PG 결제창 표시
- AC-2: 결제 성공 시 '주문이 완료되었습니다' 메시지와 주문번호 표시
- AC-3: 결제 실패 시 '결제에 실패했습니다. 다시 시도해주세요' 에러 메시지 표시
- AC-4: 결제 완료 후 주문 내역 페이지로 이동

---

#### US-106: 회원가입 [REQ-109]
> As a **일반 사용자**, I want to **회원가입하다**, So that **개인화된 서비스를 이용할 수 있다**

**Acceptance Criteria:**
- AC-1: 이메일, 비밀번호, 이름 입력 필드 제공
- AC-2: 이메일 형식이 아닐 경우 '유효한 이메일을 입력하세요' 에러 표시
- AC-3: 비밀번호 8자 미만 시 '비밀번호는 8자 이상이어야 합니다' 에러 표시
- AC-4: 회원가입 성공 시 '회원가입이 완료되었습니다' 메시지 표시 후 로그인 페이지로 이동

---

#### US-107: 로그인 [REQ-109]
> As a **일반 사용자**, I want to **로그인하다**, So that **내 정보와 주문 내역에 접근할 수 있다**

**Acceptance Criteria:**
- AC-1: 이메일, 비밀번호 입력 필드 제공
- AC-2: 로그인 성공 시 메인 페이지로 이동
- AC-3: 로그인 실패 시 '이메일 또는 비밀번호가 일치하지 않습니다' 에러 표시
- AC-4: 로그인 상태가 세션 동안 유지됨

---

### 운영 관리자

#### US-201: 상품 등록 [REQ-107]
> As a **운영 관리자**, I want to **상품을 등록하다**, So that **사용자가 구매할 수 있다**

**Acceptance Criteria:**
- AC-1: 필수 항목(상품명, 가격, 원산지) 입력 시 등록 버튼 활성화
- AC-2: 상품 이미지 최대 5장 업로드 가능
- AC-3: 등록 성공 시 '상품이 등록되었습니다' 메시지와 함께 상품 목록으로 이동
- AC-4: 중복 상품명 입력 시 '이미 등록된 상품명입니다' 경고 표시

---

#### US-202: 상품 수정 [REQ-107]
> As a **운영 관리자**, I want to **상품 정보를 수정하다**, So that **최신 정보를 유지할 수 있다**

**Acceptance Criteria:**
- AC-1: 상품 목록에서 '수정' 버튼 클릭 시 수정 폼 표시
- AC-2: 가격, 재고, 설명 등 모든 필드 수정 가능
- AC-3: 수정 성공 시 '상품 정보가 수정되었습니다' 메시지 표시
- AC-4: 필수 항목 미입력 시 저장 버튼 비활성화

---

#### US-203: 상품 삭제 [REQ-107]
> As a **운영 관리자**, I want to **상품을 삭제하다**, So that **판매 중단된 상품을 관리할 수 있다**

**Acceptance Criteria:**
- AC-1: 상품 목록에서 '삭제' 버튼 클릭 시 확인 다이얼로그 표시
- AC-2: '삭제하시겠습니까?' 확인 후 삭제 진행
- AC-3: 삭제 성공 시 '상품이 삭제되었습니다' 메시지 표시
- AC-4: 삭제된 상품은 사용자에게 표시되지 않음

---

#### US-204: 상품 목록 조회 [REQ-107]
> As a **운영 관리자**, I want to **상품 목록을 조회하다**, So that **전체 상품 현황을 파악할 수 있다**

**Acceptance Criteria:**
- AC-1: 상품명, 가격, 재고, 등록일이 테이블 형태로 표시
- AC-2: 페이지당 20개 상품 표시, 페이지네이션 제공
- AC-3: 상품명으로 검색 가능
- AC-4: 가격순, 등록일순 정렬 가능

---

#### US-205: 라벨 데이터 등록 [REQ-108]
> As a **운영 관리자**, I want to **라벨 데이터를 등록하다**, So that **OCR 스캔 시 매칭될 수 있다**

**Acceptance Criteria:**
- AC-1: 라벨 이미지 업로드 필수
- AC-2: 연결할 상품 선택 필수
- AC-3: 라벨 텍스트 데이터 입력 (브랜드명, 제품명, 빈티지 등)
- AC-4: 등록 성공 시 '라벨이 등록되었습니다' 메시지 표시

---

#### US-206: 라벨 데이터 수정 [REQ-108]
> As a **운영 관리자**, I want to **라벨 데이터를 수정하다**, So that **빈티지 변경 등을 반영할 수 있다**

**Acceptance Criteria:**
- AC-1: 라벨 목록에서 '수정' 버튼 클릭 시 수정 폼 표시
- AC-2: 라벨 이미지 교체 가능
- AC-3: 연결 상품 변경 가능
- AC-4: 수정 성공 시 '라벨 정보가 수정되었습니다' 메시지 표시

## P2 - Should Have (6개)

### 일반 사용자

#### US-108: 리뷰 작성 [REQ-106]
> As a **일반 사용자**, I want to **구매한 제품에 리뷰를 작성하다**, So that **다른 사용자에게 도움을 줄 수 있다**

**Acceptance Criteria:**
- AC-1: 구매 완료된 상품에만 리뷰 작성 버튼 표시
- AC-2: 별점 (1-5점) 선택 필수
- AC-3: 리뷰 내용 10자 이상 입력 시 등록 버튼 활성화
- AC-4: 리뷰 등록 성공 시 '리뷰가 등록되었습니다' 메시지 표시

---

#### US-109: 리뷰 조회 [REQ-106]
> As a **일반 사용자**, I want to **제품 리뷰를 조회하다**, So that **구매 결정에 참고할 수 있다**

**Acceptance Criteria:**
- AC-1: 제품 상세 페이지에서 리뷰 목록 표시 (최신순 기본)
- AC-2: 각 리뷰에 작성자 닉네임, 별점, 작성일, 내용 표시
- AC-3: 전문가 리뷰는 '전문가' 뱃지와 함께 상단에 표시
- AC-4: 리뷰 페이지네이션 (페이지당 10개)

---

### 전문가 리뷰어

#### US-110: 전문가 리뷰 작성 [REQ-106]
> As a **전문가 리뷰어**, I want to **전문가 리뷰를 작성하다**, So that **소비자에게 전문적인 정보를 제공할 수 있다**

**Acceptance Criteria:**
- AC-1: 전문가 계정으로 로그인 시 '전문가 리뷰 작성' 버튼 표시
- AC-2: 리뷰 항목: 향, 맛, 품질, 가성비 각각 5점 척도 평가
- AC-3: 전문가 리뷰는 '전문가 인증' 뱃지와 함께 표시
- AC-4: 전문가 리뷰 등록 시 관리자 승인 후 게시

---

### 운영 관리자

#### US-207: 라벨 DB 현황 조회 [REQ-102]
> As a **운영 관리자**, I want to **라벨 DB 현황을 확인하다**, So that **데이터 커버리지를 파악할 수 있다**

**Acceptance Criteria:**
- AC-1: 총 등록 라벨 수 표시
- AC-2: 상품별 라벨 매핑 현황 표시
- AC-3: 라벨 없는 상품 목록 필터링 가능
- AC-4: 최근 등록된 라벨 5개 표시

---

#### US-208: 회원 목록 조회 [REQ-109]
> As a **운영 관리자**, I want to **회원 목록을 조회하다**, So that **회원 현황을 파악할 수 있다**

**Acceptance Criteria:**
- AC-1: 이메일, 이름, 가입일, 주문 수가 테이블로 표시
- AC-2: 페이지당 20명 표시, 페이지네이션 제공
- AC-3: 이메일 또는 이름으로 검색 가능
- AC-4: 가입일순, 주문 수순 정렬 가능

---

#### US-209: 회원 정보 수정 [REQ-109]
> As a **운영 관리자**, I want to **회원 정보를 수정하다**, So that **회원 문의에 대응할 수 있다**

**Acceptance Criteria:**
- AC-1: 회원 상세에서 '수정' 버튼 클릭 시 수정 폼 표시
- AC-2: 이름, 연락처 수정 가능 (이메일 수정 불가)
- AC-3: 회원 상태 변경 가능 (활성/비활성)
- AC-4: 수정 성공 시 '회원 정보가 수정되었습니다' 메시지 표시

---

#### US-210: 재고 관리 [REQ-107]
> As a **운영 관리자**, I want to **재고를 관리하다**, So that **품절 상황을 방지할 수 있다**

**Acceptance Criteria:**
- AC-1: 상품별 현재 재고 수량 표시
- AC-2: 재고 10개 이하 상품 '재고 부족' 경고 표시
- AC-3: 재고 수량 직접 수정 가능
- AC-4: 재고 0인 상품은 사용자에게 '품절' 표시

## Ambiguities (확인 필요 사항)

### q1: OCR 정확도 목표 [REQ-112]
- **원문**: "높은 정확도 보장 (구체적 수치는 POC에서 확정)"
- **질문**: OCR 정확도 목표 수치는? (예: 90%, 95%, 99%)
- **제안**: 90% 이상, 95% 이상, 99% 이상
- **영향**: 성능 요구사항 정의 및 POC 성공 기준 설정 필요

### q2: 전문가 리뷰어 선정 기준 [REQ-106]
- **원문**: "전문가 리뷰 지원"
- **질문**: 전문가 리뷰어는 어떻게 선정/인증되는가? 내부 직원인가, 외부 전문가인가?
- **제안**: 내부 직원만, 외부 전문가 초청, 인증 심사 후 등록
- **영향**: 전문가 계정 관리 기능 범위 결정 필요

### q3: AI 비용 예산 [REQ-113]
- **원문**: "AI 토큰 비용 최적화"
- **질문**: 예상 월간 스캔 횟수는? AI 호출 비용 예산은?
- **제안**: 월 1,000회 이하, 월 10,000회, 월 100,000회
- **영향**: RAG vs AI 호출 비율 설계 및 비용 산정 필요

### q4: 카페24 연동 실패 시 대응 [REQ-121]
- **원문**: "카페24 기존 시스템과 연동 테스트"
- **질문**: 카페24 연동 실패 시 신규 개발 범위는? 기존 상품/주문 데이터 마이그레이션 필요한가?
- **제안**: 데이터 마이그레이션 포함, 신규 시스템 독립 운영, 병행 운영 기간 필요
- **영향**: POC 실패 시 본 개발 범위 및 일정 변동
