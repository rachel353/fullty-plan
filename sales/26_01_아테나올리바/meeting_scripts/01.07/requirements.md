# Client Requirements - 01.07 Meeting

## 핵심 기능 요구사항

### 1. OCR 라벨 스캔

#### [REQ-101] 올리브오일 라벨 스캔 기능
- 카메라로 올리브오일 라벨 촬영 시 제품 정보 자동 인식
- 비비노(Vivino) 와인 스캔과 유사한 사용자 경험 제공
- 라벨 DB와 매칭하여 제품 정보 표시

#### [REQ-102] 라벨 DB 구축 및 관리
- 현재 약 100~1,000개 라벨 데이터 지원
- 향후 글로벌 확장 시 최대 만 개 이내 지원
- 신규 제품 라벨 추가 기능 (관리자)

#### [REQ-103] OCR 결과 기반 제품 매칭
- 스캔 결과와 DB 매칭
- 매칭 실패 시 AI 보조 분석으로 정확도 향상
- RAG(레그) 기반 검색 우선 → AI 보조 분석 보완

### 2. 커머스 플랫폼

#### [REQ-104] 상품 조회 및 상세 정보
- 스캔된 제품 상세 정보 표시
- 가격, 원산지, 품질 정보 등 제공

#### [REQ-105] 장바구니 및 결제 시스템
- 상품 장바구니 담기
- 결제 연동 (PG사 연동)
- 구매 플로우 완성

#### [REQ-106] 리뷰 시스템
- 사용자 리뷰 작성 및 조회
- 전문가 리뷰 지원
- 제품별 리뷰 집계 및 표시

### 3. 관리자 기능

#### [REQ-107] 상품 DB 관리
- 상품 추가/수정/삭제
- 가격 및 재고 관리

#### [REQ-108] OCR 라벨 데이터 관리
- 신규 라벨 등록
- 라벨 데이터 업데이트 (빈티지별 등)

#### [REQ-109] 회원 관리
- 회원 가입/로그인 시스템
- 회원 정보 관리

## 비기능 요구사항

#### [REQ-111] OCR 응답 속도
- 라벨 스캔 결과 3초 이내 응답 (계약서에 보장 조건 명시)
- 기존 9-10초 → 3초 이내로 개선

#### [REQ-112] OCR 정확도
- 높은 정확도 보장 (구체적 수치는 POC에서 확정)
- RAG + AI 보조 분석 조합으로 정확도 향상

#### [REQ-113] AI 토큰 비용 최적화
- RAG 우선 검색으로 AI 호출 최소화
- 운영 비용 절감 구조 설계
- 스캔 횟수별 예상 비용 사전 안내

#### [REQ-114] 서버 안정성
- 24시간 서비스 가용성
- 서버 다운 시 긴급 대응 체계 (주말/야간 포함 협의)

## 통합 요구사항

#### [REQ-121] 기존 카페24 연동 테스트
- 현재 카페24 기반 시스템과 OCR 연동 테스트
- 연동 성공 시 기존 시스템 활용
- 연동 실패 시 신규 코드 개발 진행

#### [REQ-122] PG사 결제 연동
- 결제 시스템 연동
- 결제 오류 시 긴급 대응 필요

## UI/UX 요구사항

#### [REQ-131] 심리스한 스캔 경험
- 비비노 수준의 자연스러운 스캔 UX
- 사진 촬영 → 결과 표시 → 구매 연결 플로우

#### [REQ-132] 웹앱 형태 유지
- 기존 앱스토어 계정 활용 (업데이트 배포)
- 네이티브 앱 별도 개발 불필요 (현 단계)

## 비즈니스 로직 요구사항

#### [REQ-141] 단계별 개발 구조
- POC → 카페24 연동 테스트 → 본 개발 순차 진행
- 페이즈별 결과 확인 후 다음 단계 진행

#### [REQ-142] 유지보수 체계
- 1개월 무상 유지보수
- 이후 유상 (연간 20% 또는 시간제 선택)
- 주말/야간 긴급 대응 별도 협의
