# 상세 기능명세서

## 0. 공통 시스템 (모든 사용자 공통)

### 1. 회원 인증 및 지갑 관리 시스템 (User & Wallet) – 가변성 중

**1.1 일반/소셜 회원가입 및 로그인 프로세스**

- **목적:** 서비스 이용 및 재화(Ink) 소유권 증명을 위한 사용자 식별
- **화면:**
    - 로그인/회원가입 페이지 (이메일 폼, 카카오/구글 소셜 버튼)
    - 본인인증 및 약관 동의 화면
    - 프로필 설정 (닉네임 중복 확인 필수)
- **기능:**
    - **회원가입/로그인:** JWT 기반 이메일 인증 및 OAuth 2.0 소셜 연동
    - **닉네임 설정:** 작품 내 크레딧(Credit) 표기를 위한 고유 닉네임(Unique Key) 검증 및 저장
    - **본인 인증 연동:** 결제 및 정산 처리를 위한 실명 인증 모듈 연동 (NHN)

**1.2 Ink 지갑 및 트랜잭션 대시보드**

- **목적:** 유료 재화 ‘Ink’의 보유량 확인 및 투명한 사용 내역 관리
- **화면:**
    - 마이페이지 내 ‘내 지갑’ 탭
    - 보유 Ink 잔액 표시 카드
    - 트랜잭션 내역 리스트 (날짜, 유형, 금액, 관련 에피소드)
- **기능:**
    - **잔액 조회:** 유저 DB 내 `current_ink_balance` 조회
    - **트랜잭션 로깅:** 충전, 사용(제안/지지), 반환(Drop), 정산 등 모든 재화 흐름에 대한 불변 로그 기록 및 조회

## 1. 독자(Reader) 시점 기능

### 2. 작품 탐색 및 인터랙티브 뷰어 (Discovery & Viewer) – 가변성 상

**2.1 작품 탐색 홈 및 리스트**

- **목적:** 후원하거나 개입할 작품을 효과적으로 노출
- **화면:**
    - 메인 홈 (추천/인기 배너)
    - 작품 리스트 (썸네일, 제목, 작가, 현재 총 모금액)
    - 검색 및 필터 영역
- **기능:**
    - **리스트 정렬:** 최신순, 총 Bounty 금액 순 정렬
    - **검색:** 작품명, 작가명 기반 DB 쿼리 검색
    - **상태 표시:**
        - 현재 ‘후원/제안 받기(OPEN)’ 상태인 에피소드 강조
        - 최종본 공개 완료 상태(PUBLISHED) 에피소드 표시

**2.2 에피소드 뷰어 및 다이내믹 크레딧**

- **목적:** 콘티 열람 및 완성 기여자에 대한 보상 표시
- **적용 범위:** 에피소드 상태가 ‘OPEN’인 경우에만 적용
- **화면:**
    - 콘텐츠 뷰어 (이미지/텍스트 스크롤)
    - 하단 고정 액션 바 (제안하기, 후원하기 버튼)
    - 최하단 [Sponsored by] 크레딧 섹션
- **기능:**
    - **콘티 렌더링:** 작가가 업로드한 이미지/텍스트 출력
    - **다이내믹 크레딧 생성(Auto-Generation):** 완성 원고 업로드 시, 후원 데이터(Backing)를 기반으로 HTML/CSS 섹션 자동 생성
    - **Main Producer 강조:** 기여 금액(Amount) 상위 3인 별도 UI 강조
    - **리스트 정렬:** 채택된 제안자(Proposer) 우선 표시 후, 금액순 정렬
    - **영구 박제 로직:** 생성된 크레딧 데이터를 에피소드 데이터와 함께 영구 저장

**2.3 최종본 공개 이후 열람 단계 (Read-only Viewing Phase) - (2번 질문 추가 내용)**

- **목적:** 창작 및 후원 과정이 종료된 에피소드를 일반 웹툰/웹소설 플랫폼과 동일한 방식으로 제공하고, 후원 참여자에게는 기여에 대한 성취감과 보상 경험을 제공

**상태 전환 조건:**

- 작가가 최종 원고 업로드 완료
- 하나 이상의 Proposal이 `DEAL` 상태로 확정

**동작 방식:**

- 에피소드 상태가 `PUBLISHED`로 전환
- 후원 참여 여부와 관계없이 **모든 사용자 열람 가능**
- 본 단계는 2.2와 동일한 에피소드 뷰어 화면을 사용하되, 상태에 따라 UI 및 기능이 달라짐

**화면:**

- 최종 원고 전용 뷰어 (읽기 전용)
- 제안하기 / 지지하기 / 후원 버튼 **비노출**
- 하단 고정 크레딧 영역 유지

**기능:**

- **일반 열람 모드:**
    - 제안하기 / 지지하기 / 후원하기 등 개입 UI 비활성화
    - 콘텐츠 소비 중심 UX 제공
- **후원자 크레딧 노출:**
    - 2.2에서 생성된 다이내믹 크레딧을 그대로 노출
    - Main Producer 및 Sponsored by 영역 유지
- **데이터 고정:**
    - 크레딧 정보는 에피소드 데이터와 함께 영구 저장되어 이후 변경 불가

### 3. 스토리 인터벤션 로직 (Intervention Logic) – 가변성 상

**3.1 제안(Proposal) 생성 및 에스크로 처리**

- **목적:** 독자가 스토리에 개입하기 위해 Ink를 걸고 제안 생성
- **화면:**
    - 제안 작성 모달 (내용 입력, 초기 베팅액 설정)
- **기능:**
    - **최소 금액 검증:** 시스템에 설정된 고정 최소 금액(예: 100 Ink) 이상인지 확인 (동적 계산 제외)
    - **제안 생성 트랜잭션:** Proposal 객체 생성 (`description`, `proposer_id`, `seed_money`)
    - **선결제 에스크로:** 제안자의 Ink 차감 → 시스템 에스크로 계정으로 이동 (Pending 상태)

**3.2 제안 지지(Backing) 및 리스트**

- **목적:** 타인의 제안에 Ink를 더해 채택 확률을 높임
- **화면:**
    - 제안 목록 리스트 (뷰어 하단)
    - 제안 상세 카드 (내용, 모금액, 지지자 수, 지지하기 버튼)
- **기능:**
    - **지지(Backing) 트랜잭션:** 참여자 Ink 차감 → 에스크로 이동 및 Proposal의 `total_bounty`, `backer_count` 증가 업데이트
    - **제안 정렬:** 복잡한 점수 계산 없이 **'총 모금액(Total Bounty) 순'** 또는 **'최신순'**으로 단순 정렬

**3.3 직접 후원 (Direct Backing)**

- **목적:** 제안 없이 순수 창작 지원
- **화면:**
    - 후원 팝업 (금액 선택)
- **기능:**
    - **즉시 후원 처리:** 에스크로를 거치지 않고 Ink 차감 후 작가의 정산 예정금으로 즉시 이동

## 2. 작가(Creator) 시점 기능

### 4. 크리에이터 스튜디오 및 결정 시스템 (Creator Studio) – 가변성 중

**4.1 에피소드 저작 및 관리**

- **목적:** 작가가 콘티를 올리고 펀딩을 시작함
- **화면:**
    - 에피소드 관리 리스트
    - 저작 에디터 (이미지 업로드, 텍스트 입력)
    - (난이도 설정 패널 제외됨)
- **기능:**
    - **업로드 및 상태 관리:** 콘티 등록 및 상태(OPEN) 변경
    - **이미지 업로드:** 멀티플 이미지 업로드 및 순서 변경

**4.2 제안 결정(Deal/Drop)**

- **목적:** 제안을 채택하여 수익을 확정하거나 거절하여 환불 처리
- **화면:**
    - 제안 관리 대시보드 (모금액 순 정렬)
    - 제안별 Deal / Drop 액션 버튼
- **기능:**
    - **Deal(채택) 프로세스:**
        - **상태 변경:** Proposal 상태 `DEAL`로 업데이트
        - **정산 이관:** 에스크로 된 Ink → 작가 `payable_balance`로 DB 트랜잭션 이동
    - **Drop(거절) 프로세스:**
        - **상태 변경:** Proposal 상태 `DROP` 업데이트
        - **자동 환불(Rollback):** 해당 Proposal의 Backing 로그를 조회하여 모든 참여자에게 Ink 100% 자동 반환

## 3. 관리자(Admin) 시점 기능

### 5. 결제 및 통합 관리자 (Payment & Admin) – 가변성 하

**5.1 Ink 결제 시스템 (PG 연동)**

- **목적:** 실제 화폐로 플랫폼 재화 구매
- **화면:**
    - Ink 충전 상점 페이지
    - PG사(토스/페이플) 결제 창
- **기능:**
    - **결제 승인 및 충전:** PG사 결제 성공 콜백 처리 및 유저 지갑 Ink 증액
    - **결제 이력 저장:** 법적 증빙을 위한 결제 원장 기록

**5.2 정산 및 콘텐츠 관리자**

- **목적:** 수익 정산 처리 및 서비스 운영
- **화면:**
    - 회원/콘텐츠 관리 페이지 (CRUD)
    - 정산 요청 목록 및 처리 페이지
- **기능:**
    - **정산 전환 로직:** 작가가 DEAL 된 회차의 완성본 업로드 시, ‘예정 정산금’ → ‘출금 가능 금액’으로 상태 자동 변경 (수수료 차감)
    - **출금 승인:** 작가의 출금 요청 건에 대해 관리자 확인 후 승인 처리 (이체 완료 마킹)
    - **콘텐츠 관리:** 유해 게시물 숨김/삭제, 악성 유저 제재 기능
